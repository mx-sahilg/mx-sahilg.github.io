VIDEO PAYWALL LOGIC

Deprecated logic (Applicable in <= 1.47 Main app)

Steps:

  1. Fetch video details
  2. Check for json object "svod". If this block is not present play the video else move to step 3.
  3. Check for string array "groupIds" in "svod" object.
  4. If this array is empty then play the video else move to step 5.
  5. If the user has any subscription and his subscribed group id is present in the array obtained in step 3, then continue playing the video. Else move to step 6.
  6. Use first group present in array from step 3 to show the Paywall UI(as shown in images)

Updated Logic (Applicable in >= 1.48 Main app)

Steps:

  1. Fetch video details
  2. Check for json object "subscriptionInfo". If this block is not present play the video else move to step 3.
  3. Check for boolean "isContentAccessDenied" in "subscriptionInfo" object.
  4. If this boolean is false then play the  video else move to step 5.
  5. Look for "contentAccess" in "subscriptionInfo", there you will find two types of arrays "svod" and "tvod". Let's assume we consider "svod" here.
  6. Use first group present in array from step 5 to show the Paywall UI(as shown in images)

VIDEO AD-FREE LOGIC

Deprecated logic (Applicable in <= 1.47 Main app)

Steps:

  1. Fetch video details
  2. Check for json object "svod". If this block is not present play the video with ads else move to step 3.
  3. Check for the string array "adfreeGroupIds" in the "svod" object.
  4. If this array is empty then play the video with ads else move to step 5.
  5. If the user has any subscription and his subscribed group id is present in the array obtained in step 3, then continue playing the video without any ads. Else move play video with ads.

Updated Logic (Applicable in >= 1.48 Main app)

Steps:

  1. Fetch video details
  2. Check for json object "subscriptionInfo". If this block is not present play the video else move to step 3.
  3. Check for boolean "isContentAdFree" in "subscriptionInfo" object.
  4. If this boolean is false then play the video with ads else move to step 5.
  5. If this boolean is true then play the video without ads.

BLOCK DOWNLOADS

Deprecated logic (Applicable in <= 1.47 Main app)

Steps:

  1. Fetch video details
  2. Check for json object "svod". If this block is not present, then allow the download of video else move to step 3.
  3. Check for the int field "blockDownloads" in the "svod" object.
  4. If this field value is 0 then allow download of video else move to step 5.
  5. If the user has any subscription then allow the download of video else show option to start purchase flow of svod pack.

Updated Logic (Applicable in >= 1.48 Main app)

Steps:

  1. Fetch video details
  2. Check for json object "subscriptionInfo". If this block is not present, then allow the download of video else move to step 3.
  3. Check for the int field "isDownloadAccessDenied" in the "svod" object.
  4. If this field value is false then allow download of video else move to step 5.
  5. Use the svod array present in downloadAccess to start the purchase flow of svod.

APP LEVEL AD-FREE EXPERIENCE

  1. Each svod group can provide two types of ad free experience, one is video level ad free experience which we discussed above. And the other one is app level ad-free experience.
  2. App level ad free experience uses URI scheme of ads which we use MxAds library.
  3.  Any ad which we don't want to show to the subscriber comes under the uri array of exclude_filter.
  4. And the ad which we want to show comes under uri array of include_filter.
  5. Usually any ad which is not present in the exclude filter will be shown to the user. But what we if want to exclude all ads under the ott tab except one. We would not like to add the uris of all ads. So we used uri schema to specify that all ott ads are excluded except ott_videoads.
  6. The reason we keep ott_videoads in include filter is because video ads behavior is dependent on the video level settings( as explained above)

How does this work?

  1. We merge this additional configuration in the original ads configuration.
  2. And we don't parse or return the ad units for paths which are part of excluded uris.
  3. Whenever a subscription start is called, we reload the ads and reparse them as per this exclude filter mechanism.  

SVOD GROUP THEME

Svod groups have their own unique themes, Mx Gold would have a gold theme, and Mx One would have a somewhat gray theme. So to make our code generic and to avoid any hardcoding this theme was designed.

We use these theme colors to implement several components like text colors, background colors, selection color, image tint etc

SVOD JOURNEY ID

  1. We can control the groups the user would see even if the api is returning all svod groups.
  2. We set the value of svod journey id on firebase and use that value to send in the get groups

COUPON APPLICATION

  1. We allow the user to apply the coupon on CYP page
  2. If the applied coupon is incorrect or not valid for the selected product then we show the error message.
  3. Right now we support two type of coupons on android application
  1. ADD ON coupon - not required to pay anything, and user would get the subscription
  2. DISCOUNT coupon - final amount of plan is discounted and user needs to pay the discounted amount.

APP-LINK PARAMS

Name of query param

Its meaning or relevance

group_id

List of group ids to show on purchase flow(also refer to "filterPack" below)

product_id

Single plan id to select(also first index of passed group array would be used)

svodJid

Journey id of purchase flow

tab_type

It means from where this deep link has been fired. Tab type would be anything from - tab, me, search, deeplink, download, downloadAction, svod_entry_point, svod_nudge

tab_name

Additional param to augment tab_type. Tab name would be anything from - downloadScreen, topScreen

action

Action states what this deep link will do. Supported action types - svod_buy, svod_active, svod_popup

filterPack

If "true" or "1" then only group_id  passed will be shown else all are ignored

coupon

Coupon code to auto apply ( will fail if coupon not valid on default selected group id and product id)

card_id

Video id

card_name

Video name

card_type

Video type

card_af_type

Video appsflyer type

card_pub_name

Video publisher name

card_content_lanf

Video content lang

callback_value

String to receive back in the event bus callback when some purchase events are fired.

purpose

Purpose would be any one of the following - buy_for_content_access, buy_for_download, buy_for_ad_free

external

If "true" or "1", then we will start Home Activity first(OnlineActivityMediaList) else not

popupType

Popup type can be - "SUBSCRIBE_NOW_POPUP", "MEMBERSHIP_EXPIRED_POPUP", "SMART_DOWNLOAD_POPUP"

initPay

If "true" or "1", we will start the payment flow immediately when purchase page opens else no

headless

If true or "1", we will start the purchase flow in headless manner(without UI)

Samples:

mxplay://mxplay.com/svod?action=svod_buy&external=1&initPay=false&headless=false&group_id=9b16980b74877eb892dc232416b24ea9

mxplay://mxplay.com/svod?action=svod_buy&external=1&initPay=false&headless=false&group_id=9b16980b74877eb892dc232416b24ea9&product_id=com.mxplay.playergold.subs.standard.quarterly
